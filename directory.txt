# All routes are imported into app.py and registered as blueprints.
# The static/ and templates/ folders follow Flask's auto-detection.
# Admin actions are protected with simple session login (no password stored).

# Summary of core functionality:
# - app.py initializes the server and registers all routes
# - register.py handles worker form, photo upload, Aadhaar validation
# - verify.py allows admin to view unverified workers and approve/reject
# - complaint.py stores complaints in DB with worker_id and reason
# - admin.py lets admin view complaints, block workers
# - utils/validation.py checks Aadhaar format (12 digits, checksum optional)
# - Tamil content is loaded using tamil_unicode.py
# - uploads/ stores images linked by worker ID
# - SQL schema stores name, phone, work type, image paths, verified status, complaints
